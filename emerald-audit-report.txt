## Recommendations

### 1. Create CSS Custom Properties
Since emerald shows strong semantic consistency, recommend creating custom properties:

```css
:root {
  --color-success: theme(colors.emerald.500);
  --color-success-light: theme(colors.emerald.100);
  --color-success-dark: theme(colors.emerald.700);
  --color-active: theme(colors.emerald.500);
  --color-accent-start: theme(colors.emerald.500);
  --color-accent-end: theme(colors.cyan.400);
}
```

### 2. Semantic Class Utilities
Create utility classes in globals.css:
```css
.success-indicator { @apply bg-emerald-500; }
.success-background { @apply bg-emerald-100 text-emerald-700; }
.active-state { @apply text-emerald-600; }
.accent-gradient { @apply bg-gradient-to-br from-emerald-500 to-cyan-400; }
```

### 3. Component Standardization
- Standardize voice/speech components to use custom properties
- Unify success indicator styling across components
- Consolidate gradient definitions for accent elements

### 4. Green vs Emerald Decision
- Decide whether to use emerald or green for success states
- Currently mixed: some use green-*, others emerald-*
- Recommend: Use emerald for brand/accent, green for pure success states
