## Project Notes

MCP Configuration Schema Decision

B) Nested by server structure:
- Clean grouping of server config + tool policies together
- Easy to see all settings for a specific server at once
- Intuitive structure for users
- Server enable/disable at top level of each server config
- Tool policies nested under each server

Schema structure:
```json
{
  "servers": {
    "server-name": {
      "command": ["node", "server.js"],
      "transport": "stdio", 
      "enabled": true,
      "tools": {
        "tool_name": "allow-session|require-approval|deny|disable|allow-project"
      }
    }
  }
}
```

Next question should focus on configuration loading/merging behavior across hierarchy levels.
